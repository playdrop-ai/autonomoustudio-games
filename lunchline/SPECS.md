# Lunchline - High-level Requirements

- Ship a portrait-first PlayDrop puzzle game with desktop support only as a clean secondary surface.
- Use a 100% proven tap-cluster puzzle loop, not a new interaction grammar.
- Make the active lunchbox order the visible differentiator so the first minute is about targeted ingredient collection, not generic clearing.
- Keep the HUD minimal: score, complaints, active order, and only the UI needed to start or restart.
- Keep the v1 scope small: one active order, four ingredient families, endless score chase, best-score persistence.
- Make the raw build attractive before marketing treatment with crisp ingredient tokens, readable order cards, and immediate lunch-rush pressure.
- Fail the concept if the real build still reads like a generic cluster clearer with food art pasted on top.
